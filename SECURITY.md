# Security Policy

## Supported Versions

Security fixes are applied on a best-effort basis to the latest code on the
default branch.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Older branches | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues,
pull requests, or discussions.

Instead, email [mohammed.b.dev@gmail.com](mailto:mohammed.b.dev@gmail.com) with:

- A clear description of the issue
- Steps to reproduce or validate it
- The affected route, feature, or component
- Any proof-of-concept, logs, or screenshots that help confirm impact
- Your suggested disclosure timeline, if relevant

You can expect an initial acknowledgement on a best-effort basis. After the
report is validated, remediation and coordinated disclosure guidance will be
shared privately.

## Scope

This policy covers vulnerabilities in the application code, bundled frontend
assets, infrastructure-facing configuration committed to the repository, and
dependency-related issues that materially affect deployed instances.
