# Support

This repository is set up for structured collaboration and maintenance. The
fastest way to get a useful response is to use the right channel.

## Best Channel by Request Type

- Bug or regression: open a GitHub issue using the bug report template
- Feature proposal: open a GitHub issue using the feature request template
- Documentation improvement: open a GitHub issue using the documentation template
- Security concern: follow the private process in [SECURITY.md](SECURITY.md)

## Before Opening an Issue

- Read the [README.md](README.md) and [CONTRIBUTING.md](CONTRIBUTING.md)
- Confirm the behavior is reproducible on the latest code from `main`
- Gather relevant logs, screenshots, reproduction steps, and environment details

## What to Include

High-quality reports are much easier to act on. Include:

- What you expected to happen
- What actually happened
- Clear reproduction steps
- Relevant environment details
- Logs, screenshots, or failing responses when available

## Response Expectations

This project is maintained on a best-effort basis. Opening an issue does not
guarantee an SLA, but well-scoped reports and pull requests are prioritized and
easier to review.
